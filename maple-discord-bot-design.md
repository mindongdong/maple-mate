# 메이플 친구 그룹용 디스코드 봇 — 설계 명세서

> 친구끼리 같은 디스코드 서버에서 캐릭터 스펙을 비교하고, 스타포스·잠재 운빨을 겨루고, 공지·썬데이 알림을 받는 봇.
> 넥슨 메이플스토리 Open API 기반. 결정론적 코드형(LLM 미사용).

---

## 1. 아키텍처

- **단일 프로세스**: 디스코드 봇(게이트웨이) + HTTP API 서버를 한 프로세스에 통합. 같은 DB·발송 로직 공유.
- **호스팅**: 자체 서버(Mac Mini).
- **중앙 DB 1개**: 봇 서버 측에 단일 운용(사용자 기기 아님). 같은 캐릭터(ocid)면 서버 간 캐시 공유.
- **스킬 = 코드 모듈**: 기능을 모듈 단위로 캡슐화하되 런타임은 순수 코드(에이전트 호출 없음).

## 2. 범위(scope) 규칙

| 대상 | 단위 |
|---|---|
| 등록 · 비교 | **서버(길드) 단위** |
| 알림 발송 | **채널 단위** |

`/등록`은 (디스코드 유저 × 서버) 1레코드. 비교는 현재 서버 등록자 대상. 알림은 명령을 친 채널로 발송.

## 3. 명령어 (평면 구조)

조회/비교는 **인자로 분기**, 인자 없으면 기본 채널 전체 비교(단 `/스펙` 예외).

| 명령어 | 권한 | 키 | 동작 |
|---|---|---|---|
| `/등록 [닉네임] [API키(선택)]` | 누구나 | — | 서버×유저 1레코드. 키 없으면 스펙류만, 키 있으면 스타포스·잠재 해제 |
| `/스펙 [유저 1~5명]` | 누구나 | 불필요 | **인자 필수**(없으면 에러). 상세 전체 모드. 최대 5명 |
| `/아이템 [부위] [대상(선택)]` | 누구나 | 불필요 | 부위 필수(드롭다운), 1부위. 대상 없으면 채널 전체 비교 |
| `/유니온 [대상(선택)]` | 누구나 | 불필요 | 비교/개별 동일 항목. 페이지네이션 |
| `/스타포스 [기간] [대상(선택)]` | 누구나 | **필요** | 기간 프리셋(기본=최근7일)+선택 날짜. 운지수·손익 비교 |
| `/잠재 [기간] [대상(선택)]` | 누구나 | **필요** | 잠재 재설정·사용 큐브·사용 메소·등업 통합 비교표 |
| `/공지알림 on/off` | 서버 관리자 | — | 채널 토글 |
| `/썬데이 on/off` | 서버 관리자 | — | 채널 토글 |

### 3.1 `/스펙`
- 항목: 전투력 · 어빌리티 · 장착 심볼 · HEXA 코어 · HEXA 스탯
- 비교/개별 모두 **상세 전체 모드**. 인자 없으면 명시적 에러("1~5명 지정"). 1명 지정 = 단일 상세.
- 출력량은 실제 디스코드 렌더 확인 후 축소/옵션화(백로그).

### 3.2 `/아이템`
- 부위 = 디스코드 choices 드롭다운, 1부위 = 1명령.
- 항목: 스타포스 · 잠재능력 · 에디셔널 잠재 · 추가옵션 · 주문서/업그레이드
- **우열 판정 안 함** — 수치 나열만, 사용자가 직접 비교.
- 부위별 적용 항목: **동적(API에 값 있으면 표시) + 정적 보정(구조적으로 불가능한 항목 강제 숨김)** 하이브리드. "0성"과 "스타포스 불가 부위" 구분.

### 3.3 `/유니온`
- 비교/개별 동일: **유니온 레벨 + 아티팩트 레벨 + 챔피언 등급 분포 카운트**(예: SSS 2 / SS 3).
- 페이지네이션 적용.

### 3.4 `/스타포스`
- 기간: 프리셋(`오늘`·`어제`·`최근7일`·`최근30일`·`이번주`·`이번달`, 기본 `최근7일`) + 선택적 시작/종료일. 상한 30일.
- 지표: **운 지수(%) = 실제 소모 / 기대 소모**, + **손익 메소(실제 − 기대)** 병기.
- 기대값: **정가 기준 고정 표**(이벤트/PC방/MVP 할인 미반영).
- 장비 레벨 보강: (A) 현재 장착 장비(캐릭터 API)에서 레벨 매칭 → (B) 장비명→레벨 매핑표 보강 → 둘 다 실패 시 그 시도는 **기대값 산출 제외**.
- 누락 시: 표에 "N건 중 M건 기준" 투명 표시 + `error_log` 적재 + 사용자에게 "제보되었습니다".

### 3.5 `/잠재`
> ⚠️ 기획 변경(potential-handoff.md D1): 원래 `/잠재`(최다 재설정 단일 아이템) + `/잠재합계`(메소·큐브 랭킹) 2분할 → **스타포스식 단일 `/잠재` 통합 비교표**로 합치고 `/잠재합계` 폐기. 잠재 지표를 캐릭터당 1행으로 모아 commands 표/부분성공 자산을 재사용.
- 데이터 소스: **두 엔드포인트 합산** — 큐브 사용 결과(`history/cube`) + 잠재능력 재설정(메소) 결과(`history/potential`). 캐시 type 둘(`cube`·`potential_reset`). (현 메타는 메소 위주, 큐브는 과거+에디셔널)
- 비교표 컬럼: `순위` · `캐릭터` · `잠재 재설정`(큐브+메소 전체 재설정 건수) · `사용 큐브`(그 중 큐브 사용 건수) · `사용 메소`(감정비+재설정비) · `등업`(도달 등급 색 뱃지) · 기간 푸터. **정렬=사용 메소 내림차순**(동률 시 사용 큐브), 최상위 사용 메소 셀 강조.
  - **사용 메소**(G2 해소, [potential_cost.py](../maple_mate/history/potential_cost.py)): API에 금액 필드 없음(실측) → `item_level`·등급 기준 단가표 역산. 두 출처 합산 — **큐브 감정비**(`floor(계수×L²/100)×100`, 121제+ 항상 부과·120제↓ 통찰력 무료 가정) + **메소 재설정비**(레벨 구간×등급, 잠재/에디 별표). 출처: 나무위키 잠재능력.
  - **등업**(등급 상승, [CONTEXT.md](CONTEXT.md)): cube+reset 합쳐 `item_upgrade_result=="성공"` 레코드를 from-등급(레어/에픽/유니크)별로 카운트. 레전드리(종착)·미상 from 은 제외. **표시는 도달(to) 등급 색 뱃지**(`/아이템` 등급 색 공유) + 횟수 — 예: 유니크→레전드리 등업은 `레전드리` 뱃지. 0건 등급 생략, 등업 0이면 `—`.
  - **단일 대상 조회(대상 1명)** 시만 보조 노출: 사용 메소 분해(재설정/감정비) · 큐브 종류 분포 · 등급별 재설정 횟수(잠재/에디) · 등업 FROM→TO 진행. 다인 비교 시 생략.
  - 기록 0건(큐브+재설정): "기록 없음"(키 미등록과 구분).
- ⚠️ **레벨 매칭 불필요**: cube·potential 레코드에 `item_level` 직접 포함(스타포스와 달리 3단 매칭·`기준건수` 컬럼 없음).
- ⚠️ **G1(등업 "성공" 의미, 잔류)**: `item_upgrade_result=="성공"`=등급 상승 가정은 `scripts/spike_potential.py` 라이브 1콜로 확정 예정(미니스파이크). 레전드리 from 제외로 엔드게임 동급 재롤은 자연 제외.
- ✅ **G2(메소 단가표, 해소)**: 큐브 감정비 공식 + 메소 재설정 단가표 확보 → `사용 메소` 컬럼 활성. 120제 이하 큐브 감정비는 통찰력(Insight) 무료 가정(`charge_under_121=False`) — 121제+ 엔드게임 메소는 정확.

### 3.6 `/공지알림`
- 대상: **공지사항 + 업데이트**(이벤트 제외).
- 폴링: 매일 **10·12·14·16·18·21시 KST**.
- 새 공지 판정: `notice_state`에 카테고리별 마지막 발송 식별자 저장. 최초 가동 시 baseline만 기록(과거 공지 미발송).
- 발송: **제목 + 링크 + 등록일**.

### 3.7 `/썬데이`
- 스케줄: 매주 **금요일 10:10 KST 1회**, 재시도 없음.
- 매칭: 진행 이벤트 title에 "썬데이 메이플" 포함 검색. 다중 매칭 = 전부 발송, 0개 = 조용히 패스.
- 중복 방지: `notice_state`에 마지막 발송 주차 기록.
- 발송: **제목 + 링크 + 이벤트 기간**.
- **공휴일 예외(수동 발송)**: 아래 HTTP API로 처리.

## 4. 수동 썬데이 발송 (HTTP API)

- 디스코드 명령어 아님. 통합 프로세스의 HTTP 엔드포인트.
- 인증: **운영자 시크릿 토큰**(`Authorization: Bearer`, `.env`).
- 바디: 제목 · 링크 · 기간(직접 입력).
- 처리: `sunday_alert` 켜진 채널 **즉시 발송** + 해당 주차 중복 마킹(자동 발송과 공유).

## 5. 데이터 모델

### ① `registration`
- PK: (`guild_id`, `discord_user_id`)
- `maple_nickname`, `ocid`(캐싱), `api_key_encrypted`(nullable), `created_at`, `updated_at`
- 서버 내 닉네임 중복 허용(MVP).

### ② `history_cache`
- PK: (`ocid`, `type`, `date`) — `type` ∈ {starforce, cube, potential_reset}
- `payload`(원본 JSON), `fetched_at`
- **ocid 기준 공유**(서버 무관). 과거 일자 = 불변 캐시, **오늘 = 5분 TTL(KST 기준)**.

### ③ `channel_settings`
- PK: (`guild_id`, `channel_id`)
- `notice_alert`(bool), `sunday_alert`(bool)

### ④ `notice_state`
- 카테고리별 마지막 발송 공지 식별자
- 썬데이 마지막 발송 주차

### ⑤ `error_log`
- `id`, `timestamp`, `command`, `guild_id`/`discord_user_id`(nullable), `target_ocid`(nullable)
- `error_type` ∈ {nexon_api, auth_invalid, timeout, rate_limit, internal, **unmatched_equipment**}
- `retry_count`, `resolved`(nullable), `detail`(미매칭 건은 장비명)
- **재시도 발생 건만 기록**(첫 시도 성공은 미기록). 별도 제보 테이블 없음.

## 6. 보안 · 운영

- **API 키**: 앱 레벨 암호화 저장(at-rest), 마스터 키는 DB 밖 `.env`.
- **운영 비밀값 전부 `.env`**: 마스터 키, 운영자 토큰, `ADMIN_CHANNEL_ID`.
- **운영자 = 개발자 본인(단일).** 운영 요약은 `ADMIN_CHANNEL_ID` 단일 채널로.
- **일일 운영 요약**: 매일 **09:00 KST**, 전날 `error_log` 집계(타입별 건수 + 미매칭 장비 상위 N개) → 운영 채널.

## 7. 횡단(cross-cutting) 규칙

- **ocid lazy 갱신**: 평소 캐싱 사용 → 조회 실패 시 닉→ocid 재조회 1회 → 갱신 후 재시도, 실패하면 "닉 변경 가능성, `/등록`으로 갱신" 안내(이후 사용자 책임). history_cache는 ocid 기준이라 닉 변경에 안전.
- **에러 처리**:
  - 부분 성공 허용(되는 유저만 표시 + 실패 사유 행).
  - "키 미등록" vs "기록 없음" 구분.
  - 넥슨 장애/타임아웃 시 1~2회 재시도 후 안내.
  - 모든 비교 명령어 `defer`(지연 응답) 의무.
  - 실패·재시도 로그 `error_log` 적재.
- **출력**: 디스코드 임베드 통일. 초과 인원 버튼 페이지네이션. `/스펙`만 5명 상한.
- **푸터(데이터 기준 시점)**: 지난 날짜 = `YYYY-MM-DD`, 오늘자 = `HH:MM 기준`.

## 8. 알려진 데이터 한계 (출력 시 명시)

- 스타포스 이력은 **장비 레벨 미제공** → 일부 시도 기대값 산출 누락 가능.
- 큐브·스타포스 금액은 **정가 기준**, PC방/MVP/이벤트 할인 미반영 → 실제 지출과 오차.
- 캐릭터 스펙·유니온·아이템은 대체로 **전일(D-1)** 기준.
- 잠재 현 메타는 **메소 재설정 위주**(큐브 내역은 과거 + 에디셔널 중심).
