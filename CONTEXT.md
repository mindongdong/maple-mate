# Maple-Mate

친구 디스코드 서버에서 메이플 캐릭터 스펙을 비교하고, 스타포스·잠재 운빨을 겨루고,
공지·썬데이 알림을 받는 봇의 도메인 언어 사전. 넥슨 메이플스토리 Open API 기반.

## Language

### 조회 데이터 분류

**스펙류 (Spec data)**:
봇 앱 키 + `ocid`로 조회되는 공개·전일(D-1) 캐릭터 데이터(전투력·장비·심볼·HEXA·유니온). 개인 키 불필요.
_Avoid_: 캐릭터 정보, 기본 데이터

**이력류 (History data)**:
개인 넥슨 API 키 소유 계정에 스코프된 시계열 데이터(스타포스·큐브·잠재 재설정). 개인 키 필요.
_Avoid_: 강화 내역, 사용 기록

### 등록 · 비교

**등록 (Registration)**:
(디스코드 유저 × 서버) 1레코드 — 한 길드 안에서 디스코드 유저를 메이플 캐릭터에 묶는 단위.
_Avoid_: 가입, 연동

**대상 (Target)**:
비교 명령이 비교하는 등록자 집합. 인자 없으면 기본값 = 현재 서버 등록자 전원.
_Avoid_: 유저, 멤버

### 운빨 지표

**운 지수 (Luck index)**:
`실제 소모 ÷ 기대 소모` (%). 100% 미만일수록 운이 좋음.
_Avoid_: 효율, 운빨 점수

**손익 메소 (Net meso)**:
`실제 소모 − 기대 소모`. 운 지수의 절대 금액 병기.
_Avoid_: 차익, 이득

**기대값 (Expected cost)**:
정가 기준 고정 표로 산출한 기대 소모. 이벤트·PC방·MVP 할인 미반영.
_Avoid_: 평균값, 예상치

### 부재 상태 (반드시 구분)

**키 미등록 (No key)**:
개인 넥슨 API 키가 없어 이력류 조회 자체가 불가능한 상태.

**기록 없음 (No record)**:
키는 있으나 조회 기간 내 해당 활동이 없는 상태.

## Relationships

- 하나의 **등록**은 한 길드 안에서 디스코드 유저 한 명을 캐릭터 하나에 묶는다 (서버 내 닉네임 중복 허용).
- **이력류**는 등록에 개인 키가 있어야 조회 가능하고, **스펙류**는 봇 앱 키만으로 조회된다.
- **운 지수**와 **손익 메소**는 실제 소모와 **기대값**의 비교에서 파생된다.
- **대상**의 기본값은 현재 서버 등록자 전원이며, `/스펙`만 예외적으로 대상 인자가 필수다.

## Example dialogue

> **Dev:** "친구가 `/스타포스`를 쳤는데 표에 안 나와요. **키 미등록**으로 처리하면 되나요?"
> **Domain expert:** "아니요 — 그 친구가 개인 키를 등록했으면 **이력류** 조회는 됩니다. 기간 내 강화를 안 했을 뿐이면 **기록 없음**이고, 키 자체가 없으면 **키 미등록**입니다. 둘은 출력에서 반드시 구분해야 해요."

## Flagged ambiguities

- **"대상"** 이 명령 인자(특정 유저 지정)와 기본 비교 집합(서버 등록자 전원) 두 의미로 쓰임 — 해결: 대상 = 비교 집합 그 자체, 인자 없으면 현재 서버 등록자 전원(`/스펙` 제외).
- **"스펙류 vs 이력류"** 는 설계 문서에 명시되지 않았으나, API 키 모델의 핵심 경계(앱 키 공개 데이터 vs 개인 키 스코프 데이터)로 이번 그릴링 세션에서 canonical화함.
