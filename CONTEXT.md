# Maple-Mate

친구 디스코드 서버에서 메이플 캐릭터 스펙을 비교하고, 스타포스·잠재 운빨을 겨루고,
공지·썬데이 알림을 받는 봇의 도메인 언어 사전. 넥슨 메이플스토리 Open API 기반.

## Language

### 조회 데이터 분류

**스펙류 (Spec data)**:
봇 앱 키 + `ocid`로 조회되는 공개·전일(D-1) 캐릭터 데이터(전투력·장비·심볼·HEXA·유니온). 개인 키 불필요.
_Avoid_: 캐릭터 정보, 기본 데이터

**이력류 (History data)**:
개인 넥슨 API 키 소유 계정에 스코프된 시계열 데이터(스타포스·큐브·잠재 재설정). 개인 키 필요.
_Avoid_: 강화 내역, 사용 기록

### 등록 · 비교

**등록 (Registration)**:
(디스코드 유저 × 서버) 1레코드 — 한 길드 안에서 디스코드 유저를 메이플 캐릭터에 묶는 단위.
_Avoid_: 가입, 연동

**대상 (Target)**:
비교 명령이 비교하는 등록자 집합. 인자 없으면 기본값 = 현재 서버 등록자 전원.
_Avoid_: 유저, 멤버

### 운빨 지표

**운빨 수치 (Luck score)**:
실제 총 소모 메소가 가능한 결과 분포에서 차지하는 행운 백분위(높을수록 운 좋음 = 싸게 끝냄). 표에는 `상위 X%`로 표시하며, **손익과 같은 메소 기반이라 정렬이 일관**되고 파괴 손해가 반영된다. 단 "상위 50%"는 중앙값(흔한 결과)이라 우편향 때문에 기댓값보다 싸다(ADR-0002).
_Avoid_: 운 지수, 효율, 성공률

**총 사용 메소 (Total meso)**:
기간 내 스타포스 시도에 쓴 메소 합(시도당 정가 비용 합산). 장비 레벨 매칭된 시도만 집계.
_Avoid_: 강화비, 소모액

**기댓값 대비 손익 (Net meso)**:
`총 사용 메소 − 기대값`. 음수면 기댓값보다 덜 써서 이득(운 좋음). 운빨 수치의 절대 금액 병기.
_Avoid_: 차익, 이득

**기대값 (Expected cost)**:
정가 기준 마르코프 기대 소모(시작★→최종★, 파괴→12성 되돌림 반영). 이벤트·PC방·MVP 할인 미반영.
_Avoid_: 평균값, 예상치

### 잠재 지표

**등업 (등급 상승, Tier-up)**:
잠재능력 등급이 한 단계 오른 사건(레어→에픽→유니크→레전드리). 큐브·메소 재설정 이력에서 `item_upgrade_result=="성공"` 레코드로 판정하며, **from-등급**(등업 직전 등급 = `before_potential_option` 최고 등급)별로 카운트한다. 레전드리는 종착이라 from 이 될 수 없고, 엔드게임의 동급(레전드리) 재롤 "성공"은 등업에서 제외된다. `/잠재` 비교표 뱃지는 **도달(to) 등급**(`/아이템`과 동일 등급 색)으로 보여준다 — 예: 유니크에서 올랐으면 `레전드리` 뱃지. 단일 대상 보조는 `유니크 → 레전드리` 처럼 출발·도달을 화살표로 함께 표기.
_Avoid_: 등급업, 승급, 잠재 성공
⚠️ "성공"=등급 상승 가정은 G1 미니스파이크(`scripts/spike_potential.py`)로 라이브 확정 예정.

**사용 큐브 (Cube used)**:
기간 내 큐브 사용 건수(`history/cube` 레코드 수). 메소 재설정(`history/potential`)과 별개로 큐브 횟수만 센다. 비교표 컬럼명 = `사용 큐브`.
_Avoid_: 총 큐브, 큐브 개수

**잠재 재설정 (Potential reset)**:
기간 내 잠재능력을 재설정한 **전체 횟수 = 큐브 사용(`history/cube`) + 메소 직접 재설정(`history/potential`)**. 큐브든 메소든 둘 다 잠재 재설정 행위라 합산한다(넥슨 API가 수단별로 두 엔드포인트로 나눌 뿐). `사용 큐브`는 그 중 큐브 분, 메소 분은 `잠재 재설정 − 사용 큐브`로 도출된다.
_Avoid_: 메소 재설정만 "잠재 재설정"으로 지칭(큐브 분 누락)

**사용 메소 (Meso spent)**:
잠재에 쓴 총 메소 = **큐브 감정비**(큐브 사용 시 레벨 기준 메소 수수료) + **메소 재설정비**(레벨 구간·등급 기준 잠재/에디 재설정 단가)의 합. API에 금액 필드가 없어 `item_level`·등급으로 단가표 역산([potential_cost.py](../maple_mate/history/potential_cost.py)). 큐브 아이템 자체 값(현금/이벤트)은 스코프 밖.
_Avoid_: 강화비, 큐브값

### 부재 상태 (반드시 구분)

**키 미등록 (No key)**:
개인 넥슨 API 키가 없어 이력류 조회 자체가 불가능한 상태.

**기록 없음 (No record)**:
키는 있으나 조회 기간 내 해당 활동이 없는 상태.

### 알림

**썬데이 메이플 (Sunday Maple)**:
주말에 강화 확률·보상 버프를 주는 넥슨의 주기성 이벤트. 넥슨 이벤트 공지(`notice-event`)에 제목으로 등장하며, 한 이벤트 배너가 여러 주에 걸쳐 진행될 수 있다.
_Avoid_: 썬데이, 주말 이벤트

**썬데이 알림 (Sunday alert)**:
주말 썬데이 메이플을 앞두고 **금요일에 미리** 보내는 사전 예고. 일요일 당일 알림이 아니다 — 친구들이 주말 강화를 준비하도록 금요일 오전에 "이번 주말 썬데이 메이플" 제목·링크·기간을 채널로 발송한다.
_Avoid_: 일요일 알림, 썬데이 발송

## Relationships

- 하나의 **등록**은 한 길드 안에서 디스코드 유저 한 명을 캐릭터 하나에 묶는다 (서버 내 닉네임 중복 허용).
- **이력류**는 등록에 개인 키가 있어야 조회 가능하고, **스펙류**는 봇 앱 키만으로 조회된다.
- **기댓값 대비 손익**(절대 금액)과 **운빨 수치**(백분위)는 둘 다 **총 사용 메소 vs 기대값**에서 파생되어 정렬이 일관된다 — 손익은 평균 대비 차액, 운빨은 분포 내 위치.
- **대상**의 기본값은 현재 서버 등록자 전원이며, `/스펙`만 예외적으로 대상 인자가 필수다.

## Example dialogue

> **Dev:** "친구가 `/스타포스`를 쳤는데 표에 안 나와요. **키 미등록**으로 처리하면 되나요?"
> **Domain expert:** "아니요 — 그 친구가 개인 키를 등록했으면 **이력류** 조회는 됩니다. 기간 내 강화를 안 했을 뿐이면 **기록 없음**이고, 키 자체가 없으면 **키 미등록**입니다. 둘은 출력에서 반드시 구분해야 해요."

## Flagged ambiguities

- **"대상"** 이 명령 인자(특정 유저 지정)와 기본 비교 집합(서버 등록자 전원) 두 의미로 쓰임 — 해결: 대상 = 비교 집합 그 자체, 인자 없으면 현재 서버 등록자 전원(`/스펙` 제외).
- **"스펙류 vs 이력류"** 는 설계 문서에 명시되지 않았으나, API 키 모델의 핵심 경계(앱 키 공개 데이터 vs 개인 키 스코프 데이터)로 이번 그릴링 세션에서 canonical화함.
- **이력류 스코프(계정) vs 집계 단위(캐릭터)** — 개인 키는 **계정 전체** 이력을 반환하나(`character_name` 이 캐릭터별로 상이), 봇의 `/스타포스`·`/잠재` 집계 단위는 **등록 캐릭터 1개**(스펙류와 동일). 해결(Phase 3 그릴링): 이력 레코드를 `character_name == 등록 닉네임` 으로 필터. 부캐 강화는 집계 제외(MVP). ⚠️ 닉 변경 시 과거 이력의 `character_name` 불일치 가능성은 잔류 리스크.
- **메소 컬럼의 "정가"** — **총 사용 메소**와 **기대값** 모두 정가 단가로 계산(이력의 `cost_discount_rate` 할인 미반영). 따라서 **기댓값 대비 손익**은 할인 타이밍(썬데이 등)에 휘둘리지 않는다. 해결(Phase 3 그릴링).
- **운빨 수치 vs 메소 손익** — 둘 다 메소 기반(운빨=가능 분포 내 백분위, 손익=평균 대비 차액)이라 정렬이 일치하고 파괴 손해가 양쪽에 반영된다(ADR-0002). 둘 다 레벨 매칭 시도만 집계 → 미매칭 장비는 양쪽에서 제외(기준건수 `M/N건`). 단 운빨의 "상위 50%"=중앙값, 손익 0=평균이라 우편향으로 중립점은 약간 다르다.
