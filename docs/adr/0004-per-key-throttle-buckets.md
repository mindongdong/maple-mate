# ADR-0004 — 넥슨 키 단위 스로틀 버킷

- **상태:** 채택 (Accepted)
- **일자:** 2026-06-10
- **관련 문서:** [deploy-plan.md](../deploy-plan.md) §3 단계 2(병목 분석), [scale-tuning-work-order.md](../scale-tuning-work-order.md)(구현 D2·3-1), [ADR-0001](0001-nexon-personal-key-model.md)(개인키 모델)
- **이력:** 스케일 튜닝 그릴링 세션(2026-06-10)에서 D2 결정으로 확정.

## 맥락 (Context)

현재 `NexonClient` 는 **단일 Lock + 단일 next_allowed**([client.py:63-86](../../maple_mate/nexon/client.py#L63-L86))로
봇의 **모든** 넥슨 호출(앱키 스펙류 + 유저별 개인키 이력류)을 한 버킷에 직렬화한다(0.25초 간격 = 4/s).
이력류 cold 조회 1건이 최대 365콜이라 **전역을 ~90초 독점**해 무관한 스펙류 명령까지 정지시킨다.

핵심 사실: **넥슨 레이트리밋은 키 단위다.** 그리고 두 키 부류의 한도가 영구히 다르다 —

- **앱키**(봇 소유 1개): 서비스단계 등록 시 500/s·2,000만/일로 상향 가능.
- **개인키**(유저 각자 발급, ADR-0001): 유저에게 서비스단계 등록을 시킬 수 없어 **개발단계 5/s·1,000/일에 고정**.

즉 "전역 스로틀 값 하나"로는 답이 없다: 0.25 유지 → 이력류가 앱키 트래픽 독점,
0.02로 상향 → 이력류 연타가 그 개인키의 5/s 를 초과해 429 백오프 유발.

## 결정 (Decision)

**스로틀을 키별 버킷으로 분리한다** — `api_key → next_allowed` dict 하나.

1. **앱키 버킷**: 간격은 환경변수 `NEXON_THROTTLE`(기본 0.25). 넥슨 서비스단계 승인 *후* 운영자가 0.02로 내린다.
2. **개인키 버킷**(키마다 독립): 간격 **0.2초(5/s) 고정 상수** — 개인키 한도는 유저가 바꿀 수 없는 넥슨 정책값이라 설정 노출이 무의미.
3. 버킷 식별은 `_request(api_key=...)` 인자 그대로(None=앱키). 프로세스 내 in-memory — **단일 인스턴스 전제**(deploy-plan §1)와 결합.

## 검토한 대안 (Alternatives Considered)

- **전역 값 상향 + config 화만** — *기각.* 가장 단순하나 이력류 연타가 개인키 5/s 를 초과 → 429 백오프 재시도로 자가회복되긴 해도 낭비적이고, 키 단위인 넥슨 한도 모델과 구조가 어긋난 채 남는다.
- **키별 버킷 + 전역 동시성 세마포어** — *기각.* 넥슨 서버 보호 관점 최대 안전이나 100서버 피크가 초당 몇 건 수준이라 세마포어가 일을 안 함. 과잉설계.
- **Redis 분산 레이트리미터** — *기각(유보).* 멀티 인스턴스에서만 필요. 단계 3(deploy-plan §3) 영역.

## 결과 (Consequences)

**긍정**
- 이력류 cold 가 **앱키 트래픽을 더 이상 막지 않음** — "전역 ~90초 정지" 시나리오 소멸.
- 개인키 5/s 를 봇이 선제 준수 → 429 백오프 왕복 예방.
- 넥슨 한도 모델(키 단위)과 코드 구조가 1:1 대응 — 서비스단계 등록 효과가 `NEXON_THROTTLE` 한 값으로 반영됨.

**부담 / 잔류**
- **이력류 cold 1년은 여전히 그 유저에게 ~73초**(365콜 ÷ 5/s) — 키별 분리는 *타인* 보호이지 본인 가속이 아니다. 본인 가속은 history_cache(이미 구현)와 prune 정책이 담당.
- **in-memory 버킷 = 단일 프로세스 전제** — 인스턴스 2개면 키당 실효 한도가 2배로 새어 429 위험. 단계 3 분산 시 Redis 리미터로 대체해야 함(이 ADR 의 만료 조건).
- 버킷 dict 는 활성 개인키 수만큼 자람(수백 entry 수준) — 정리 로직 불요로 판단.
