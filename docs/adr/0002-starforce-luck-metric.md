# ADR-0002 — 스타포스 운빨 지표: 메소 행운 백분위(운빨 수치)

- **상태:** 채택 (Accepted)
- **일자:** 2026-06-07
- **관련 문서:** [CONTEXT.md](../../CONTEXT.md)(운빨 수치·총 사용 메소·기댓값 대비 손익), [starforce-expected-value-data.md](../starforce-expected-value-data.md)(§2 비용·§5 기대값·§8 운빨 모델)
- **이력:** 같은 설계 사이클에서 두 모델을 거쳐 확정 — ① 메소 비율(운지수) → ② 성공 확률 백분위 → ③ **메소 백분위(최종)**. ②는 실데이터 테스트에서 손익과 어긋나 폐기(아래 대안).

## 맥락 (Context)

`/스타포스` 헤드라인 지표는 "누가 강화 운이 좋았나"를 한 수치로 비교·정렬한다. 두 번의 실데이터 테스트로 모델이 진화했다.

1. **운지수 = 실제 ÷ 기대 메소(%)** — 무진행 구간(시작★=최종★)에서 기댓값 0 → `—`(미정의). 폐기.
2. **성공 확률 백분위**(포아송-이항) — 확률 대비 성공 횟수의 백분위. `—` 문제는 해결했으나, **메소 손익과 심하게 어긋났다.** 실데이터(1년):
   - 라딘라면: 성공률 운 상위 14%인데 손익 +49억 / 손바: 성공률 운 상위 22%인데 손익 **+158억**. 운빨 1위와 손익 1위가 불일치.
   - 원인: (a) 손익은 **고성문 한 방**이 좌우(손바 데이브레이크 18→21★ 하나가 +103억)하는데, 성공 백분위는 모든 시도를 균등 가중해 한 방을 못 잡음. (b) **파괴→재등반이 싼 저성문 "성공"을 양산** → 성공 백분위는 올라가지만 파괴 메소는 출혈(라딘라면 블랙빈·글러브에서 −85억). 이진 성공 모델이 파괴 손해를 못 본다.
   - → 사용자 피드백: 운빨이 메소 손익과 같은 방향이어야 직관적.

## 결정 (Decision)

운빨 수치를 **메소 행운 백분위**로 정의한다 — 손익과 같은 메소 기반이라 정렬이 일관되고, 파괴 손해가 자연 반영된다.

1. 레벨 매칭된 각 아이템의 `시작★→최종★` climb 에 대해, **실제 총 메소가 가능한 결과 분포에서 차지하는 백분위**를 구한다. `L = 100 × [P(M > 실제) + ½·P(M = 실제)]` (M=시뮬 총 메소, mid-p). 싸게 끝냈으면 L 큼(운 좋음).
2. 분포는 **몬테카를로**로 추정한다. 핵심: **"성수별 시도 횟수" 분포는 레벨 무관**(확률표만 결정) → `(시작★,최종★)`별로 **1회만 시뮬·캐시**하고, 런타임엔 레벨별 비용을 곱해 합산. 결정적 시드라 재현 가능. 매 요청 재시뮬 없음.
3. 표시 **`상위 (100−L)%`**, 정렬 **L 내림차순**(운 좋은 사람 rank 1), 최상위 강조.
4. **운빨·메소 둘 다 레벨 매칭 시도만** 집계(동일 기준). 미매칭은 양쪽 제외 + `M/N건`.
5. ⚠️ **기준점 주의:** "상위 50%"는 **중앙값**(가장 흔한 결과)이다. 비용 분포는 파괴로 우편향이라 **중앙값 < 평균(기댓값)** → "기댓값대로 쓴 사람"은 상위 ~65%(평균보다 흔한 결과가 더 쌈). 손익(평균 기준)과 정렬은 일치하나 중립점은 약간 다르다.

## 검토한 대안 (Alternatives Considered)

- **A. 성공 확률 백분위(포아송-이항)** — *시도 후 폐기.* 확률 대비 성공/실패를 정확히 재고 레벨 무관·항상 정의되나, **메소 손익과 어긋남**(위 맥락). 파괴 손해 미반영. 사용자가 원한 건 "메소 운과 일치하는 운빨".
- **B. 메소 비율(실제÷기댓값)** — *기각.* 손익과 같은 숫자원이라 정렬은 일치하나 **진짜 백분위가 아님**(분포의 분산 미반영). 고분산 climb 84%와 저분산 climb 84%가 같은 운으로 보임. 사용자가 "상위 X%"를 명시 선택.
- **C. 라이브 몬테카를로(요청마다 시뮬)** — *기각.* 정확하나 매 요청 수천만 난수 → 수 초. 레벨 무관 캐시(채택)로 회피.
- **D. 분석적 평균+분산 + 정규근사** — *기각.* 시뮬 없이 빠르나 우편향 분포의 꼬리에서 부정확하고 "기댓값=50%" 앵커도 안 맞음. 캐시 MC 가 더 충실.
- **파괴 추가 가중(성공 모델 위)** — *해당 없음.* 메소 백분위는 파괴 비용을 본질적으로 반영하므로 불필요.

## 결과 (Consequences)

**긍정**
- 운빨과 손익이 **같은 메소 기반 → 정렬 일치**(손바 상위 16.8% > 라딘 35.8%, 손익도 +158억 > +49억). 사용자 혼란 해소.
- 파괴 손해가 운빨에 반영(파괴 출혈 = 비싼 실제 = 낮은 백분위).
- 캐시·결정적 시드로 빠르고 재현 가능. 무진행 구간도 측정(헛돈=불운).

**부담 / 잔류**
- **중앙값 vs 평균 기준점 차이**(결정 5) — "손익 0(=평균대로)"이 "상위 50%(=중앙값)"와 안 맞고 상위 ~65%. 정렬 방향은 일치하므로 실사용 혼란은 작으나 문서화 필요.
- **레벨 의존:** 메소 백분위는 레벨이 있어야 산출 → 전부 미매칭이면 운빨 `—`(성공 모델은 레벨 무관이었음). 레벨 자동 학습(별도)으로 커버리지 확보.
- **소표본:** climb 적으면 백분위 출렁임(재미 지표). MC 표본 2000.
- **MC 워밍업:** 새 `(시작★,최종★)` 쌍 첫 등장 시 시뮬 비용(캐시 후 즉시). 친구 그룹은 쌍이 한정적.
