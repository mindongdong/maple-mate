# ADR-0001 — 넥슨 개인 키 모델: 이력류는 유저별 개인 키, 스펙류는 봇 앱 키

- **상태:** 채택 (Accepted)
- **일자:** 2026-06-04
- **근거 검증:** [Spike 0 실호출 검증](../api-verification-plan.md) — 판정 **GO**
- **관련 문서:** [CONTEXT.md](../../CONTEXT.md)(스펙류/이력류·키 미등록/기록 없음), [maple-discord-bot-design.md](../../maple-discord-bot-design.md)(§1 등록·§5 데이터모델·§8 한계), [work-plan.md](../work-plan.md)

## 맥락 (Context)

넥슨 메이플스토리 Open API는 봇이 쓰는 데이터를 두 종류로 나눈다.

- **스펙류** — 캐릭터/유니온/장비/심볼/HEXA. 봇 **앱 키 + `ocid`** 로 **임의의 공개 캐릭터**를 전일(D-1) 기준 조회. 개인 키 불필요.
- **이력류** — 스타포스·큐브·잠재 재설정 `history/*`. **`ocid` 파라미터가 없고**, 응답이 **API 키를 소유한 넥슨 계정에 스코프**된다.

따라서 "봇 앱 키 하나로 친구들의 강화 이력을 조회한다"는 설계는 **API 구조상 불가능**하다. 이력류를 쓰려면 각 유저가 자신의 넥슨 개인 키를 등록해, 그 키로 그 유저 본인 이력만 조회해야 한다. 이 가정은 봇의 핵심 데이터 모델(유저별 키 저장·암호화·부분 성공 UX)을 좌우하므로, 코딩 전에 **Spike 0**으로 실측 검증했다.

### Spike 0 실측 근거 (2026-06-04, GO)

- 개인 키로 최근 7일 이력 호출 → 반환된 **15건(스타포스 5 + 큐브 10) 전부 키 소유 계정 캐릭터**, 타 계정 0건.
- `history/*` 엔드포인트는 `count`·`date`·`cursor`만 받고 **`ocid` 파라미터를 받지 않음**.
- 잘못된 키 → `400 OPENAPI00005`("The apikey is not valid."), 활동 없음 → `200` + 빈 배열(`count:0`) → **"키 무효"와 "기록 없음"이 명확히 구분**됨.
- (한계) 검증 키가 앱 키 = 개인 키 동일 값이라 "앱 키로 타인 이력 조회 불가"의 음성 대조는 미실험. 단 키 스코프 메커니즘(ocid 부재 + 키 소유 계정만 반환)은 확정.

## 결정 (Decision)

1. **이력류 명령(`/스타포스`·`/잠재`·`/잠재합계`)은 개인 키를 등록한 유저만 사용**한다. 키로 그 유저 본인 계정 이력만 조회한다.
2. **스펙류 명령(`/스펙`·`/유니온`·`/아이템`)·알림은 봇 앱 키만으로 전원 조회**한다(공개 D-1 데이터).
3. **`/등록`에서 개인 키 입력은 선택**이다. 키 없이 등록 = 스펙류만, 키 등록 = 이력류까지 해제.
4. 등록 시 받은 개인 키는 **Fernet으로 암호화하여 at-rest 저장**하고, 마스터 키는 DB 밖 `.env`에 둔다(design §6).

## 검토한 대안 (Alternatives Considered)

- **A. 봇 앱 키 하나로 모든 유저 이력 조회** — *기각.* `history/*`에 `ocid`가 없고 키 소유 계정에 스코프되므로 **구조적으로 불가능**(Spike 0 확정). 가능했다면 키 저장·암호화·부분성공 로직이 전부 불필요했을 것이므로, 이 대안의 기각이 이 ADR의 핵심이다.
- **B. OAuth/넥슨 계정 연동** — *해당 없음.* 넥슨 Open API는 발급형 API 키 기반이며 서드파티 OAuth 위임 흐름을 제공하지 않는다.
- **C. 유저가 매 조회 시 키를 직접 입력** — *기각.* 디스코드 채팅에 키 평문 노출 + 반복 입력 UX 열악. 1회 등록 + 암호화 저장이 우월.

## 결과 (Consequences)

**긍정**
- 이력류가 키 소유자 본인 데이터로 한정 → 프라이버시·정확성 확보, 타인 이력 무단 조회 불가.
- 스펙류는 키 없이도 전원 비교 가능 → 진입 장벽 낮음(키 없는 친구도 `/스펙` 참여).

**부담 / 후속 처리 필요**
- **개인 키 = 민감정보** → Fernet 암호화 저장, 마스터 키 `.env` 관리(누출 시 회전).
- **"키 미등록"(키 없음) vs "기록 없음"(키 있고 기간 내 활동 없음)** 을 출력에서 반드시 구분(CONTEXT.md).
- **부분 성공:** 이력 비교 명령은 키 등록 유저만 표에 포함하고, 미등록 유저는 별도 표시.
- **키 라이프사이클:** 무효(`OPENAPI00005`)·만료·삭제·갱신 대응(design §8 미해결 항목).
- **rate limit:** Spike 0에서 `test_` 키는 `x-ratelimit-limit: 5`(~5/sec) 관측. 유저별 키라 부하는 분산되나 클라이언트 스로틀·재시도 필요. 운영(live) 키 한도는 별도 확인 대상.
- **ocid lazy 갱신:** 스펙류는 ocid 캐싱 + 실패 시 1회 재조회(design §6). 이력류는 ocid 무관(키 스코프).
